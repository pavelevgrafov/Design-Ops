# knowledge/ — сайдкар «Знания»

База доказательств пайплайна: одна заметка = один источник. Обычный
Markdown с frontmatter — **Obsidian является рекомендованным
просмотрщиком, не зависимостью**: vault открывается любым редактором.

## Структура

- `index.yaml` — компактный индекс для агента (≤4 KB): id, уровень,
  тезис, теги. **Генерируется валидатором, рукой не правится.**
- `sources/` — заметки источников (сид — правила бан-листа и чек-листов).
- `decisions/` — зеркало decision log проекта.
- `gates/` — артефакты гейтов (выборы, обоснования).
- `briefs/` — concept briefs (экспорт для клиента).

## Дисциплина (проверяется knowledge-validate.py)

1. Правило чек-листа без заметки не существует; заметка-сирота
   удаляется (каждая заметка цитируется минимум одним правилом).
2. Уровни доказательности: `research` > `industry-standard` > `curated`.
   Уровня `opinion` нет.
3. `verified_at` старше 12 месяцев → заметка к перепроверке.
4. Агент читает только индекс; полные заметки — по id, по требованию
   (бюджет контекста).
