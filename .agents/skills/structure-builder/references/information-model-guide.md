# Information Model Guide — 5 patterns (v5.1)

Pick exactly ONE pattern by the primary job. The pattern defines what earns
`data-priority="p1"` and the order of blocks. Do not blend patterns on one
screen; different screens of one flow may use different patterns (record it).

## 1. answer-first — the user came for an answer

- **When:** docs, FAQ, support, news, knowledge bases, recipes.
- **P1 rule:** the answer itself, or the shortest path to it, in the first
  viewport. Navigation and cross-links are p2; related content p3.
- **Skeleton signature:** answer block top-left, TOC/search immediately
  adjacent; no hero.
- **Failure mode:** burying the answer under a welcome banner or hero image.

## 2. task-first — the user came to do a thing

- **When:** SaaS, tools, dashboards, forms, admin panels, banking.
- **P1 rule:** the primary action control and its minimum viable context
  (current status, last result). Secondary actions p2, settings p3.
- **Skeleton signature:** action surface dominates; nav is a thin rail;
  empty state teaches the first task.
- **Failure mode:** a dashboard of vanity widgets while the actual task
  needs three clicks.

## 3. object-first — the user is evaluating an object

- **When:** e-commerce, portfolio, real estate, cars, courses, hotels.
- **P1 rule:** the object's decisive evidence — key visual + 2–3 deciding
  attributes + price/verdict anchor. Specs p2, reviews/similar p3.
- **Skeleton signature:** media block + fact panel above the fold; long-form
  below.
- **Failure mode:** a poetic intro pushing the object's facts below the fold.

## 4. event-first — the user is deciding whether to attend

- **When:** conferences, concerts, webinars, meetups, openings.
- **P1 rule:** what/when/where + the one reason to care + ticket/CTA. Program
  and speakers p2, venue/sponsors/FAQ p3.
- **Skeleton signature:** date-place-CTA triad pinned to the first viewport.
- **Failure mode:** an atmospheric video hero with no date visible.

## 5. comparison-first — the user is choosing among alternatives

- **When:** pricing pages, vs-pages, product finders, ranked reviews.
- **P1 rule:** the comparison unit (table/cards) with the deciding criteria
  visible without scrolling. Methodology p2, individual details p3.
- **Skeleton signature:** N columns, identical structure, one recommended
  option marked structurally (not visually — that is K2).
- **Failure mode:** a marketing story before the comparison itself.

## Choosing fast

Ask: "What does success look like 30 seconds after arrival?" — got an answer
(1), did the thing (2), judged the thing (3), decided to attend (4), picked an
option (5). If two compete, the primary_job wins; the loser becomes p2.

## Pattern → Gate 1 question

Tell the user which pattern you chose and why in one sentence ("This is a
plan-selection page, so the comparison comes first"). Gate 1 approves
structure INCLUDING the pattern; a pattern change later = structural change →
K1 delta.
